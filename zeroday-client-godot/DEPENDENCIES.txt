ZeroDayMMO - Dependencies
=========================

REQUIRED DEPENDENCIES
---------------------

1. Godot Engine 4.0+
   - Game engine for running scenes
   - Install: sudo apt install godot3 (Ubuntu/Debian)
   - Install: sudo dnf install godot (Fedora)
   - Install: sudo pacman -S godot (Arch)
   - Download: https://godotengine.org/download

2. .NET SDK 8.0+
   - Required for C# script compilation
   - Install: sudo apt install dotnet-sdk-8.0 (Ubuntu/Debian)
   - Install: sudo dnf install dotnet-sdk-8.0 (Fedora)
   - Install: sudo pacman -S dotnet-sdk (Arch)
   - Download: https://dotnet.microsoft.com/download

OPTIONAL DEPENDENCIES
---------------------

3. Git
   - For version control
   - Install: sudo apt install git (Ubuntu/Debian)
   - Install: sudo dnf install git (Fedora)
   - Install: sudo pacman -S git (Arch)

4. Mono (if not using .NET SDK)
   - Alternative C# runtime
   - Install: sudo apt install mono-complete (Ubuntu/Debian)

SYSTEM REQUIREMENTS
-------------------

- OS: Linux (Ubuntu 20.04+, Fedora 35+, Arch Linux)
- RAM: 4GB minimum, 8GB recommended
- GPU: OpenGL 3.3 compatible graphics card
- Disk: 2GB free space minimum
- CPU: Dual-core processor or better

QUICK INSTALL
-------------

Run the automated installer:

    ./install_dependencies.sh

Or use the launcher with install flag:

    ./launcher.sh --install

MANUAL INSTALLATION
-------------------

Ubuntu/Debian:
    sudo apt update
    sudo apt install godot3 dotnet-sdk-8.0

Fedora:
    sudo dnf install godot dotnet-sdk-8.0

Arch Linux:
    sudo pacman -S godot dotnet-sdk

VERIFY INSTALLATION
-------------------

Check Godot:
    godot --version

Check .NET SDK:
    dotnet --version

Build project:
    cd /root/ZeroDayMMO/zeroday-client-godot
    dotnet build

TROUBLESHOOTING
---------------

Problem: "godot: command not found"
Solution: Install Godot or add to PATH

Problem: "dotnet: command not found"
Solution: Install .NET SDK or add to PATH

Problem: Build errors
Solution: Run 'dotnet clean && dotnet build'

Problem: Scene won't load
Solution: Check Godot console output for errors

For more help, see LAUNCHER_README.md
